How It Works

nCaptcha employs multiple detection layers to identify automated browsers:

1. Interaction Heuristics

Analyzes mouse movement patterns, acceleration, jitter, and timing. Bots typically exhibit perfect linear movements or erratic jumps that humans don't produce. Tracks events and pointer pressure data.

2. Browser Fingerprinting

Captures complete window object properties including enumerable status, native function signatures, and prototype chains. Detects modifications made by Playwright, Puppeteer, Selenium, and Patchright.

3. Automation Markers

Checks for telltale signs and other framework-specific properties that automation tools inject.

4. CDP Domain Detection

Exploits known bugs in Patchright where Console CDP Domain is disabled. Tests console event capture, page error events, and service worker registration.

5. Property Comparison

Compares thousands of browser properties against baseline from legitimate browsers. Detects missing properties, type mismatches, and altered descriptors.
